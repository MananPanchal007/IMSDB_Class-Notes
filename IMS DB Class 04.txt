IMS DB CLASS 04:-
=========================================================================

CALL 'CBLTDLI' --

SYNTAX:
		CALL 'CBLTDLI'  USING  1. DLI-FUNCTION
							2. PCB-MASK
							3. IO-AREA
							4. SSA.


1. DLI-FUNCTION:			GET CALL'S		--> SEQ -GN (GET NEXT)  - RANDOM -> GU (GET UNIQUE) -> GNP (GET NEXT PARENTAGE)
													-> HOLD**  GHN , GHNP , GHU		>READ + LOCK + THEN LATER UPDATE

					INSERT CALL		--> ISRT
					REPLACE CALL		--> REPL
					DELETE CALL		--> DELT
	

PROCOPT:		 GRID ALL O P


	GET CALL ->  PROCOPT IN THE PSB -->  G , GO , GOP , A			=G- GET ,  O - ONLY  , P - PATH  ,  A- ALL ACCESS.
	ISRT CALL -> PROCOPT IN THE PSB -->  I ,  A				- I -INSERT
	REPL CALL	-> PROCOPT  IN THE PSB -->  GR ,  A				- G -GET  ,  R - REPL , A - ALL
	DELT CALL -> PROCOPT IN THE PSB ->  GD , A				- G -GET , D - DELT , A - ALL

		
		CALL 'CBLTDLI' USING  'GU'		-> INVALID SYNTAX;

	01 DLI-GU	  PIC    X(04)  VALUE 'GU  '.
	01 DLI-GN	  PIC    X(04)  VALUE 'GN  '.
	01 DLI-ISRT	  PIC    X(04)  VALUE 'ISRT'.

		CALL 'CBLTDLI' USING DLI-GN		--> VALID SYNTAX


2. PCB-MASK:		
			
	01 PCB-MASK.
		03 DATABASENAME     PIC		X(08).		--> DATABASE WE ARE ACCESS THRU THIS PCB
		03 SEGEMENT-LEVEL	  PIC	X(02).		--> SEGEMENT WHICH THE CONTROL IS NOW ACCESS
		03 STATUS-CODE	  PIC    X(02).		-CONTAINS THE RETURN CODE OF THE DLI CALL
		03 PROCOPT		  PIC    X(04).			-> AS PER PSB
		03 FILLER			  PIC   S9(05) COMP.
		03 SEGEMENT-NAME	  PIC    X(08).			= SEGEMENT WHICH DLI IS USING
		03 KEYLEN			  PIC   S9(05) COMP.
		03 LEVEL			  PIC   S9(05) COMP.
		03 CONCATED-KEY-LEN  PIC   X(N).			-- SUM OF THE KEYLEN - PROVIDED IN PSB**

STATUS-CODE		-> 2 CHAR STATUS CODE.

			>>  BLANK  -> '   '		-> SUCCESSFUL
			
				GET CALL ->   	1. GB  -> END OF THE DATABASE
								2. GE	-> RECORD NOT FOUND
								3. GG**  -> ON GU CALL -> CONTENTION ISSUE (DEAD LOCK)

				ISRT		->   II  --> DUPLICATE RECORD
					PSB - SSA - OPEN -->   A*    AJ* - SSA INVALID



3. IO-AREA  -> INPUT OUTPUT AREA.
	
		GET CALL -> OUTPUT TO CATCH THE DATA
		ISRT CALL -> INPUT TO SEND THE DATA

	--> FOR ANY CALL -> WE ONLY ACCESS 1 SEGEMENT DATA** (1 RECORD)

			
		CALL 'CBLTDLI'   USING  DLI-GN,
							PCB-MASK,
							ORGSEG-IO.

	--> FOR SPECIAL CALL = PATH CALL**  -> WE WILL BE TRYING TO GET MULTIPLE SEGEMENT DATA IN SINGLE CALL.
			
			-> WE NEED TO SPACE TO CATCH ALL SEGEMENT IN SINGLE IO-AREA.
OPTION-1
		01 IO-AREA.
			COPY ORG
			COPY CLI
			COPY ACCT
OPTION-2
		COPY ORG
		COPY CLI
		COPY ACCT
		
		66 IO-AREA  RENAMES FST-VAR-ORG THRU LAST-VAR-ACCT.
		

4. SSA -> SEARCH SEGEMENT ARGUMENT*  (LIKE UR WHERE CONDITION)

		-> IF USED - FOR EACH SEGEMENT ONLY 1 SSA IS ALLOWED.


		CALL STATEMENT:		2 TYPES		1. UNQUALIFIED CALL		--> WITHOUT SSA		1. GN CALL  2. REPL
										2. QUALIFIED CALL 		--> WITH SSA			-> ALL 

		QUALIFIED CALL WITH SSA ->  2 TYPES:		1. UNQUIFIED SSA	--->   HERE WE PROVIDE ONLY SEGEMENT NAME**
											2. QUALIFIED SSA	-->   HERE WE PROCIDE SEGEMENT NAME + OCCURANCE VALUE (KEY VALUE)


	UNQ-SSA:			01  UNQ-SSA		PIC		X(09)  VALUE 'SEGEMENT '.		SEGEMENT - 8 CHAR  , 9TH POS - BLANK

	Q-SSA			01  Q-SSA.
						03 SEGEMENT-NAME		PIC	X(08)  VALUE 'SEGMENT-NAME'.
						03 FILLER				PIC	X(01) 	VALUE '('.
						03 FEILD-NAME			PIC  X(08)	VALUE 'FLDNAME'.		-DEFINED IN THE DBD
						03 OPER				PIC	X(02)	VALUE 'EQ'.
						03 FIELD-VALUE		PIC  X(N)     VALUE SPACE.						N - AS PER THE KEY LENGTH
						03 FILLER				PIC  X(01)	VALUE ')'.


				ORGSEG01(ORGCODE EQ '12345')


	--> 1. SSA   2. DLI  2. SEGEMENT COPYBOOK 	4. PCB MASK

			CALL 'CBLTDLI'	DLI-GU,			-->  PATH CALL => 'D'
							PCB-MASK-1,
							IO-AREA-SEGEMENT,	==> SEGEMENT AREA
							QUAL-SSA-ORG,			ORGCODE = 12344 && ...
							QUAL-SSA-CLEINT,		CLIECN = 123
							QUAL-SSA-ACCT.		ACCT=98767			==>STATUS-CODE = '  '  OR  'GE' -RECORD NOT FOUND
